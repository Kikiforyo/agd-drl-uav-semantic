# experiment_brief.md

## 1. 文档目的
本文件用于向 Codex 提供论文实验的最小但完整上下文，避免 Codex 在实现时自行猜测系统设定。

## 2. 论文任务概述
目标是实现一个**全双工多天线 UAV 隐蔽语义通信系统**，在满足隐蔽性硬约束的前提下，通过 UAV 轨迹控制与波束相位控制，提升语义信息传输性能。

系统关键词：
- FDB-CUSC
- DeepSC
- MINE
- 闭式功率反推
- AGD-DRL
- 多头注意力
- 条件扩散策略
- 双 Critic

## 3. 系统参与者
### Alice
- 固定地面发送端
- 发送私密文本序列
- 文本经语义编码与信道编码后映射为复数符号流

### UAV
- 带内全双工（IBFD）
- 配备多天线阵列
- 一边接收 Alice 的语义上行，一边向 Willie 发射人工噪声
- 需要尽量降低人工噪声泄漏造成的自干扰

### Willie
- 移动监测者
- 用于检测是否存在无线传输行为
- 位置随时间变化，可建模为参考路径 + 高斯-马尔可夫扰动

## 4. 研究问题
本文不是单纯的功率分配或轨迹优化问题，而是以下多个机制的联合优化：
1. 语义信息传输
2. 隐蔽通信约束
3. 全双工自干扰抑制
4. 多天线方向性人工噪声
5. UAV 轨迹控制
6. 高维连续相位控制

## 5. 方法设计
### 5.1 语义模块
- 采用 DeepSC 风格结构
- 包括：
  - semantic encoder
  - channel encoder
  - channel decoder
  - semantic decoder
- 在基础验证阶段可冻结
- 在主性能对比阶段必须纳入完整系统，并允许联合微调

### 5.2 语义性能估计
- 使用 MINE 或兼容互信息下界估计器
- 用于近似语义信息传输效率
- 可用于定义 semantic rate / SUDT / outage 等指标

### 5.3 控制变量
本文主方法中，agent 不直接输出全部变量。

对 AGD-DRL / TD3 / DDPG：
- 输出：
  - UAV 速度 / 位移控制量
  - 多天线相位向量
- 不直接输出：
  - 干扰功率
- 干扰功率由 KL 隐蔽性硬约束闭式反推得到，并受最大功率限制裁剪

### 5.4 AGD-DRL
- 使用多头注意力（MHA）提取动态对抗环境中的关键状态特征
- 使用条件扩散策略生成高质量连续动作
- 使用双 Critic 提高训练稳定性
- 目标是在高维连续动作空间中缓解探索不足与局部最优问题

## 6. 实验阶段划分
### 6.1 基础验证实验
目的：验证关键物理机制正确性。

允许：
- 冻结 DeepSC / MINE / 语义模块
- 使用物理层代理指标或简化语义指标
- 使用启发式 baseline

建议验证内容：
1. 闭式功率反推是否能在 KL 约束下稳定工作
2. 方向性波束是否降低满足隐蔽要求所需干扰功率
3. 空间零陷是否改善 UAV 接收链路质量
4. 简化系统下 reward / gamma_u / P_jam / proxy semantic throughput 的趋势是否合理

### 6.2 主性能对比实验
目的：验证完整系统最终性能。

要求：
- 使用完整系统：DeepSC + MINE + 控制策略
- 主性能阶段不允许继续把语义模块冻结成纯控制问题
- 所有对比算法共享相同预训练语义初始化
- 进入主实验后，各算法都进行联合微调

推荐流程：
1. 先预训练语义模块，保存公共初始化
2. 对各算法加载同一套初始权重
3. 在同一环境设置下进行联合训练
4. 用统一评估脚本输出主图和主表

## 7. 对比算法
### 主对比
- AGD-DRL
- TD3
- DDPG

这三者应尽可能保持同维连续动作空间：
- 轨迹控制
- 连续相位控制
- 干扰功率闭式反推

### 参考基线
- DRL-JPPO

说明：
- DRL-JPPO 原始文献是位置与功率联合优化，不是本文的连续相位控制方法
- 实现时可作为参考基线纳入，但需要在文档中明确指出其与本文方法并非完全同构

## 8. 评价指标
建议统一输出以下指标：
- cumulative SUDT
- per-slot semantic rate
- outage ratio
- KL violation ratio
- covert feasible ratio
- average gamma_u
- average Gamma_w 或 Willie 检测代理量
- average P_jam
- max P_jam
- reward curve

## 9. 主图要求
论文主性能对比至少生成四类图：
1. 算法收敛图
2. 累积语义吞吐量对比图
3. 隐蔽性对比图
4. 干扰功率对比图

建议同时保存：
- PNG
- PDF
- 绘图原始 CSV

## 10. 公平性约束
所有主性能对比算法必须共享：
- 相同环境参数
- 相同随机种子集合
- 相同训练预算
- 相同评估 episode 数
- 相同语义模块初始化
- 相同日志与绘图脚本

不允许：
- 为个别算法单独增加训练轮数
- 为个别算法单独修改环境难度
- 为个别算法偷偷替换评价指标
